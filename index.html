Phase 1 — アカウント・環境準備
01
LINE公式アカウントを作成する
LINE Developers · Messaging API
FREE ▼

LINE公式アカウントとMessaging APIチャネルを作成します。無料プランで月1,000通まで送信可能です。

LINE公式アカウント開設ページにアクセスし、アカウントを作成する（個人LINEアカウントで登録可）
LINE Developers Consoleにログインし、プロバイダーを作成
「Messaging APIチャネル」を作成 → チャネルシークレットとチャネルアクセストークン（長期）を発行・メモしておく
（任意・推奨）「LINEログインチャネル」も作成する。友だち追加の流入経路追跡・UUID管理に必要
💡 LINEログインチャネルがないと流入追跡・マルチアカウント統合が機能しません。後から設定も可能ですが、最初から作ることを推奨します。
02
Cloudflareアカウントを作成する
Cloudflare Workers · D1 Database
FREE ▼

サーバー代0円の秘密はCloudflareの無料枠です。5,000友だちまでこれだけで運用できます。

Cloudflareにアクセスし、無料アカウントを作成
Wrangler CLI（Cloudflare開発ツール）をインストール：Node.js（v18以上）が必要
# Node.jsのバージョン確認（v18以上が必要）
node --version

# Wrangler CLIをグローバルインストール
npm install -g wrangler

# Cloudflareにログイン
wrangler login
📊 Cloudflare無料枠：Workers 10万リクエスト/日、D1 5GB。5,000友だち規模なら余裕で収まります。
Phase 2 — LINE Harnessのデプロイ
03
リポジトリをクローンしてインストール
GitHub · pnpm · Node.js
REQUIRED ▼

GitHubからソースコードを取得し、依存関係をインストールします。pnpmが必要です。

# pnpmがない場合はインストール
npm install -g pnpm

# リポジトリをクローン
git clone https://github.com/Shudesu/line-harness-oss.git
cd line-harness-oss

# 依存パッケージをインストール
pnpm install
04
D1データベースを作成する
Cloudflare D1 · SQLite · 42テーブル
FREE ▼

Cloudflare D1（SQLiteベース）にデータベースを作成し、LINE Harnessのスキーマ（42テーブル）を適用します。

# D1データベースを作成（出力されるdatabase_idをメモ）
npx wrangler d1 create line-crm

# スキーマを適用（42テーブルが作成される）
npx wrangler d1 execute line-crm \
  --file=packages/db/schema.sql
上記コマンドで表示される database_id を apps/worker/wrangler.toml に記入する
05
シークレットキーを設定してデプロイ
LINE API Keys · Cloudflare Secrets
API KEY ▼

STEP 1でメモしたLINEのAPIキーをCloudflare Workersのシークレットとして登録し、デプロイします。

# LINEのAPIキーをシークレットとして登録
npx wrangler secret put LINE_CHANNEL_SECRET
npx wrangler secret put LINE_CHANNEL_ACCESS_TOKEN
npx wrangler secret put API_KEY  # 任意の文字列でOK

# LINEログインチャネルも使う場合（推奨）
npx wrangler secret put LINE_LOGIN_CHANNEL_ID
npx wrangler secret put LINE_LOGIN_CHANNEL_SECRET

# Cloudflare WorkersにデプロイするにはWorkerディレクトリで実行
cd apps/worker
npx wrangler deploy
⚠️ API_KEY は外部からのアクセスを守る重要なキーです。推測されにくいランダムな文字列（32文字以上推奨）を設定してください。
デプロイ成功後、Worker URL（例：https://your-worker.workers.dev）が表示される。これをメモする
LINE Developers ConsoleのWebhook URLに https://your-worker.workers.dev/webhook を登録し、Webhookの利用をONにする
Phase 3 — Claude Code連携
06
管理画面を起動して動作確認
Next.js 15 · localhost:3001
REQUIRED ▼

Next.js製の管理画面をローカルで起動し、デプロイしたWorker APIに接続して動作を確認します。

# プロジェクトルートに戻る
cd ../../

# WebアプリのAPI URL環境変数を設定
echo 'NEXT_PUBLIC_API_URL=https://your-worker.workers.dev' \
  > apps/web/.env.local

# 管理画面を起動（http://localhost:3001 で開く）
pnpm dev:web
ブラウザで http://localhost:3001 を開く
STEP 5で設定した API_KEY を入力してログイン
ダッシュボードが表示されれば正常に接続完了 🎉
07
Claude CodeにMCPサーバーを追加する
MCP Server · 17 Tools · 自然言語操作
CORE ▼

これが最大の目玉。Claude CodeのMCPに LINE Harness を追加することで、チャットの自然言語だけでLINE運用の全操作が可能になります。

Claude Codeを起動し、/mcp コマンドでMCP設定を開く
以下の設定を追加する（claude_desktop_config.json または .mcp.json）
{
  "mcpServers": {
    "line-harness": {
      "command": "npx",
      "args": ["@line-harness/mcp-server"],
      "env": {
        "LINE_HARNESS_API_URL": "https://your-worker.workers.dev",
        "LINE_HARNESS_API_KEY": "YOUR_API_KEY"
      }
    }
  }
}
Claude Codeを再起動して設定を反映
